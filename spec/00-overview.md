# Megatest Specification

## What is Megatest?

Megatest is a visual regression testing SaaS (hosted at megatest.dev) with an open-source self-hosting option. It detects unintended visual changes in web applications by comparing screenshots across commits.

**How it differs from CLI-based visual testing tools:** Megatest is an external service, not a test runner inside your repo. It auto-discovers your application's workflows via an AI agent — users don't write config at all. Config lives in a Git repository (a dedicated config repo by default, or the project repo itself) and is AI-generated, not hand-written. Everything else — baselines, diffs, approvals, history — lives in the SaaS.

## Core Flow

```
1. Developer pushes a commit or opens a PR
2. GitHub sends a webhook to Megatest
3. Megatest clones the repo, spins up the app in a Docker container
4. Playwright on the worker host runs the configured workflows,
   taking screenshots at defined checkpoints
5. Screenshots are compared against baselines using pixelmatch
6. Results are posted back: commit status + PR comment with a review link
7. Reviewer opens the link, sees diffs side-by-side, approves or rejects
8. Approved screenshots become the new baselines
```

## Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Browser automation | Playwright | Industry standard, CSS/role/text/testid selectors, maintained by Microsoft, huge ecosystem |
| Config authoring | AI-generated (.megatest/ directory) | Configs are generated by an AI discovery agent, not hand-written |
| Config format | Multiple YAML files in .megatest/ | Broken into chunks for AI digestibility |
| Element targeting | Playwright locators (CSS, role, text, testid, label) | Flexible targeting via page.locator(), page.getByRole(), page.getByText(), page.getByTestId(), page.getByLabel() |
| Image diffing | pixelmatch | Standard, fast, pure JS, battle-tested |
| Auth model | Multi-tenant GitHub OAuth | Users log in via GitHub, access scoped to their repos |
| Worker isolation | Docker per run | User code runs in containers; browser runs on host |
| Git provider | GitHub only (MVP) | GitLab is a future extension |
| Job queue | BullMQ + Redis | Reliable, supports concurrency controls and retries |
| Database | PostgreSQL | Simplifies deployment, required for concurrent SaaS operation |
| API framework | Fastify | Fast, schema validation, plugin ecosystem |
| Web UI | Embedded SPA (vanilla JS or Preact) | Served by API server, no separate build for MVP |
| Deployment model | Hosted SaaS + open-source self-hosting | megatest.dev as primary, Docker Compose for self-hosted |
| Config storage | Git repository (config repo) | Single model: separate config repo by default, or same repo as project. Avoids needing write access to the project repo. |
| LLM provider | Configurable (Claude, OpenAI, etc.) | Provider abstraction; users bring their own API key. Best for open-source. |
| Trigger configuration | Server-side project settings | Operational config, not test definition; avoids bootstrap problem |
| Pricing | Free tier + usage-based paid | Free for small projects (500 screenshots/month, 3 projects), paid per screenshot/run |

## Project Structure

```
megatest/
  package.json
  spec/                          # ← You are here

  server/
    index.mjs                    # Fastify entry point
    routes/
      webhooks.mjs               # POST /api/webhooks/github
      auth.mjs                   # OAuth flow
      projects.mjs               # CRUD /api/v1/projects
      runs.mjs                   # /api/v1/runs
      checkpoints.mjs            # /api/v1/checkpoints, approvals
      discoveries.mjs            # /api/v1/discoveries
    middleware/
      auth.mjs                   # Session auth guard
      webhook-verify.mjs         # HMAC signature verification
    db/
      schema.sql                 # Database schema
      index.mjs                  # Connection + query helpers
    storage/
      index.mjs                  # Storage abstraction
      local.mjs                  # Local filesystem
      s3.mjs                     # S3-compatible

  worker/
    index.mjs                    # BullMQ processor entry point
    docker.mjs                   # Docker container lifecycle
    clone.mjs                    # Git clone with installation token
    config-parser.mjs            # .megatest/ directory parser
    setup.mjs                    # In-container environment setup
    runner.mjs                   # Workflow execution engine
    browser.mjs                  # Playwright wrapper
    comparator.mjs               # pixelmatch-based screenshot diff
    notifier.mjs                 # GitHub status + PR comment posting

  discovery/
    index.mjs                    # Discovery job processor
    agent.mjs                    # LLM-powered exploration agent
    detector.mjs                 # Setup auto-detection (package.json, etc.)
    generator.mjs                # Config file generation from discoveries

  ui/
    index.html                   # SPA shell
    app.mjs                      # Router + main app
    components/
      dashboard.mjs              # Project list
      project.mjs                # Project detail + run history
      review.mjs                 # Run review page (primary interface)
      checkpoint.mjs             # Single checkpoint viewer
    styles.css

  lib/
    config.mjs                   # Config schema validation
    github.mjs                   # GitHub API client (App auth, installation tokens)
    utils.mjs                    # Shared utilities
```

## Implementation Phases

### Phase 1: Core Pipeline
PostgreSQL, clone, Docker, Playwright, screenshot, pixelmatch.

### Phase 2: GitHub Integration
GitHub App, OAuth, webhooks, commit statuses, PR comments.

### Phase 3: Review UI
Review page, dashboard, approval workflow.

### Phase 4: Discovery Agent
AI-powered exploration, config generation, auto-discovery.

### Phase 5: SaaS Platform
Org model, quotas, trigger rules, billing.

### Phase 6: Route Detection + Polish
Framework parsers, auto re-discovery, cleanup.
