# Megatest Instructions

Project-specific instructions for the /megatest skill.
This file is automatically maintained — edit freely, and the skill will also update it when it learns something new.

## Project Quirks

- This repo IS megatest itself. The `.megatest/` config tests the **serve UI webapp**.
- The serve UI is fed by **isolated fixture data** in `test-fixtures/serve-data/.megatest/` — this keeps the demo report stable regardless of megatest runs.
- A separate serve instance must be running on port 3001 before running tests:
  ```
  node projects/serve/bin/megatest-serve.js --config serve.config.test.yml
  ```
- The demo fixture data is generated by `scripts/generate-test-report.cjs`. Regenerate with:
  ```
  node scripts/generate-test-report.cjs
  ```
- The production serve instance at megatest.opendle.com serves real projects — do NOT use that as the base URL.
- The review page URLs use project name `megatest-demo` (matching the test serve config).

## Timing

- The serve UI renders synchronously so 500ms wait after navigation is sufficient.
- Tab switching (Differences/New/Passed) is instant — 300ms is enough.
